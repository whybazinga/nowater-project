# Android students program

## Цели

```
Изучить возможности SDK и получить теоретические знания об основных
компонентах приложения и подходах
Получить практический опыт разработки несложного мобильного
приложения
Научиться работать с системой контроля версий
```
## Программа

```
Вводное занятие. Android как платформа. Настройка среды разработки.
Создание проекта, структура.
Введение в основные компоненты Android-приложения. Основные
компоненты интерфейса. View. Введение в списки. Логирование. Toasts.
Activity и его жизненный цикл. Android manifest. Понятие контекста.
Интент. Broadcast receiver.
Разметка графического интерфейса. Правила правильной разметки.
Ресурсы приложения: графические ресурсы, строковые ресурсы, анимации.
Хранение данных: SharedPreferences. Файлы. Работы с БД в Android.
Разработка приложение со сложным UI. Material design. Списки, Side menu,
диалоги, уведомления, тосты.
Многопоточность в Android.
Работа с интернет в Android. HTTP. XML. JSON. Use service like https://my-
json-server.typicode.com/ to create API with emulated data.
Работа с сервисами в Android.
Custom view.
Подведение итогов. Общие темы из жизни Android-разработчика.
Подготовка к собеседованиям.
```
```
По предмету проводится дифференцированный зачет. Это значит, что за курсовой
проект каждый будет получать оценку.
```
## Требования к приложению для сдачи

```
Требования к приложению для сдачи приведены ниже.
```
```
4 - разработать функциональное приложение по своей теме с 3 экранами и
переходами между ними. Экраны должны содержать функционирующие
кнопки и поля ввода, которые управляют логикой приложения. Логика
приложения должна состоять не менее чем из 10 методов (кроме методов
ЖЦ activity). Основные этапы работы приложения должны быть отображены
в LogCat;
5 - добавить поддержку локализаций (при смене локализации в настройках
устройства язык приложения должен меняться), различных конфигураций
экрана (9-patch, разные картинки для разной плотности и размера, xml-
drawables), plurals. Для взаимодействия с пользователем применить Toast и
диалоги;
6 - реализовать список с элементами произвольной разметки;
7 - реализовать хранение настроек приложения в Shared Preferences,
данных - в БД. Выполнять операции записи/чтения в отдельном потоке;
8 - реализовать получение данных со стороннего web-сервиса, обработать
данные в XML/Json формате и использовать их для отображения в
приложении;
9 - реализовать выполнение длительных операций в Service.
```
```
Игры также в общем случае должны соответствовать требованиям выше.
Например, пункты для получения оценок 6 и 7 будут выполнены, если
реализовать экран рекордов.
```
